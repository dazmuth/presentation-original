Lab 4 (Overview): Creating Cloud Site

Estimated time for completion: 25-35 minutes
Activities to be completed:
  • Build a VPC site in F5XC which build an AWS VPC
  • View how AWS Availability Zones are defined
  • View how Instances are created in AWS from F5XC
      
Lab Scenario

With Namespace, Key Pair, Virtual Site, and Cloud Credentials configured, create an AWS Virtual Private Cloud (VPC) Customer Edge with a private DNS address from F5XC. Later labs will create Load Balancers that allow connectivity to this VPC via the subnet defined in this lab. Define a public ssh key, which is used in later labs (via a TCP Load Balancer and its attached Origin Pool) to connect to the AWS instance via port 22. In training, the largest AWS instance type (m5.4xlarge) is used to ensure maximum performance, but in a production environment you may want to use a smaller instance type to minimize costs.

INSTRUCTOR ACTIONS/CALLOUTS:

For step 2 Create Labels:
  • The Key Pair must be selected in the Labels field so that the corresponding virtual site is attached to the VPC, causing the pods to be built in that virtual site
For step 3 Select Site Type and VPC Parameters:
  • Feel free to toggle to AWS and show an existing VPC ID to demonstrate that if a student wants to use an existing VPC in production, the VPC must match exactly
For step 4 Select Ingress/Egress:
  • Deployments with an extensive internal network in addition to an internet presence may want to use Ingress/Egress Gateway (Two Interface). See the link at this step in the student guide for information
  • Be sure students don't select Availability Zone us-east-1e or us-east-1f as these have caused issues in the training environment
For step 5 Define Subnet information:
  • Any subnet within 172.31.0.0/16 can be used. Those that are listed here are intended to make it easy to keep track of who's using what
For step 7 Define Site Node Parameters:
  • Be sure students paste the entire ssh key, including "ssh-rsa" at the beginning of the code
  • If the incorrect AWS Instance Type for Node is selected, Destroy the existing VPC and clone a new one with a new name (see instructions in step 7). This will delete the associated AWS VPC
For step 8 Instantiate the Instance:
  • Toggle to AWS and select Instances. Notice that the VPCs are being constructed, with each named "master-0." Once its Instance State is Running, check F5XC and verify that its status is Applied. For each instance, perform the following:
    • Click the checkbox for that instance to display details below (ensure the Details tab is displayed)
    • Locate the VPC ID in the details, which contains the student ID (studentX)
    • Click the little edit box to the right of each instance to rename it (since every instance by default is named "master-0")
    • Rename it to studentX
For step 9 Allow DNS and SSH services
  • The AWS Tags field should allow the assigning of labels to VPC deployments, which then appear in AWS. But as of June 2023, this feature isn't working as expected.
